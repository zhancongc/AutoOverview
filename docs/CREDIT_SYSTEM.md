# 积分系统设计文档

## 💰 定价模型

### 当前大模型调用成本分析

生成一篇综述需要：**6-13 次大模型调用**

| 阶段 | 调用次数 | 说明 |
|------|----------|------|
| 生成大纲 | 1次 | 生成综述框架 |
| 生成综述 | 3-10次 | Function Calling 迭代 |
| 统计分析 | 1次 | 生成统计信息 |
| 自然总结 | 1次 | 生成语言总结 |

**平均：约 10 次调用**

## 🔄 积分流转

### 充值流程
```
1. 用户选择套餐
2. 创建订单
3. 发起支付（微信/支付宝）
4. 支付完成回调
5. 增加用户积分
6. 记录交易
```

### 消费流程
```
1. 用户发起操作（生成综述）
2. 检查积分余额
3. 扣除积分
4. 记录交易
5. 执行操作
6. 操作失败则退还积分
```

## 💳 数据模型

### 积分套餐 (credit_packages)
```sql
- id: 套餐ID
- name: 套餐名称
- credits: 积分数量
- price: 价格
- description: 描述
```

### 积分交易 (credit_transactions)
```sql
- id: 交易ID
- user_id: 用户ID
- amount: 变动数量（正/负）
- balance_after: 交易后余额
- transaction_type: 类型（purchase/consume/reward/refund）
- description: 描述
- related_id: 关联ID
```

### 充值订单 (credit_orders)
```sql
- id: 订单ID
- order_no: 订单号
- user_id: 用户ID
- package_id: 套餐ID
- amount: 积分数量
- amount_paid: 实付金额
- payment_status: 支付状态
- payment_method: 支付方式
```

## 📈 收入预测

### 假设场景
- 日活用户：100人
- 转化率：10%（10人付费）
- 人均消费：¥69.9（标准包）

```

¥699 × 30天 = ¥20,970/月
```

```
¥20,970 × 12月 = ¥251,640/年
```

## ⚠️ 风险控制

### 防刷机制
```
1. 同一设备/IP 限制注册
2. 短信验证码限流（60秒1次）
3. 新用户限制提现
4. 异常交易监控
```

### 退款策略
```
1. 充值后7天内未消费可全额退款
2. 已消费部分按原价扣除
3. 退款后积分清零
```
